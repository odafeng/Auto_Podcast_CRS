假設你今晚下班,吃完飯,坐在電腦前,打開了 Google Chrome——然後愣在那邊,不知道要在網址列打什麼。上一集我講了一堆「為什麼要學」,但你現在需要的不是更多說服,你需要有人告訴你具體打開哪一個東西、敲哪幾行字。這集就是那個東西。我們就從今晚開始。

先說工具。第一週你只需要一個東西:Google Colab。網址是 colab.research.google.com。就這樣,你有 Google 帳號,你就能用。不需要裝任何軟體,不需要設定 Python 版本,不需要碰 terminal。你打開那個網頁、登入、點 New notebook,你的 Python 環境就在那邊等你了。

為什麼是 Colab 而不是很多教學文章說的那個 Anaconda?我來講。Anaconda 是很好的東西,但你裝 Anaconda 的那個晚上,你有至少三分之一的機率會卡在環境設定——找不到某個 path、Python 版本衝突、conda 跑了十分鐘沒動靜然後報一個你看不懂的 error。你會花一個小時搞這件事,最後什麼 code 都沒寫到。第一週的目標不是把環境搞好,第一週的目標是讓你跑到你的第一行 code。Colab 幫你跳過那個坑。pandas、numpy、sklearn 全部預裝好在裡面,你不需要自己 install。你的 notebook 存在 Google Drive,換電腦打開一樣在。這就是為什麼第一週用 Colab。

好,在我講 code 之前我要說一件很重要的事,請你把剛才拿著的東西放下來認真聽一下。

Colab 是 Google 的雲端服務。[emphasizes] 帶有病人資料的 Excel,絕對不能上傳進去。不管你覺得那份資料已經去識別化到什麼程度,醫院的資料使用合約,幾乎普遍都禁止把任何與病人相關的資料放到外部的雲端平台。Google 的伺服器不在醫院 firewall 裡面。這不是我在嚇你,這是真的會有法律和倫理問題的事情。

[pauses]

所以第一週練習,你要用的是開放資料,或者你自己編造的假資料。Kaggle 上有一個叫 Titanic 的 dataset,那是鐵達尼號乘客的生還資料,完全免費、完全公開、跟病人完全無關。或者 pandas 這個套件裡面有內建的 iris dataset,也是傳統的練習資料。用這些。等你真正要分析你手上的病人資料,那個時候你要在自己的電腦、在院內的環境裡跑——那件事情怎麼設定,我們下一集再說。今晚先把 Colab 打開、先把 Titanic 資料讀進來。

現在說 code。我知道你可能預期我會叫你寫 print("Hello World"),但那個對我們沒有意義。你是醫師,你有 Excel,你想對你的資料做點什麼。所以第一個 code 不是 Hello World,是讀一個 Excel。

三行。只要三行。我口述一下大意。第一行,你引入一個叫 pandas 的工具,然後給它一個縮寫叫 pd,這樣之後打起來比較短。第二行,你告訴 Python 去讀某個 Excel 檔,把它存成一個叫 df 的東西,df 是 dataframe 的縮寫,就是一張表格的意思。第三行,你叫它把這張表格的前五行印出來給你看。就這樣。你跑完這三行,你就已經看到你的資料了。

為什麼這個比 Hello World 有意義?因為你跑完之後你看到的不是一串字,你看到的是你認識的東西——欄位名稱、數字、你知道那代表什麼的資料。那個感覺會讓你想繼續。Hello World 跑完你只會覺得,好,然後呢。

還有一件事。如果你坐在那邊,不確定那三行 code 的語法怎麼打,你不需要去 Google 一篇教學文章。你直接打開 ChatGPT 或 Claude,把你要做的事情用中文說給它聽:「我想用 Python 讀一個 Excel,印出前五行。」它會把三行 code 完整給你,你複製貼上進 Colab,按 Shift+Enter,就跑了。第一天不需要記 syntax。

[CHUNK_BREAK]

說到 ChatGPT 和 Claude,這兩個是你第一週最重要的工具。但有一個用法上的差別很重要:一次問一件事。不要問「我怎麼學 Python」,那個問題太大,它給你的回答你只能看著螢幕點頭然後什麼都不做。你要問具體的事情,比如說:「我有一個 Excel,裡面有 patient_id、age、sex 三欄,我想知道男女平均年齡有沒有統計上的差異,請給我 Python code。」這樣問,它就能直接給你可以跑的東西。把 error message 貼進去問也是一樣的邏輯,把整段紅字貼過去,告訴它你在做什麼、你想要什麼結果。

另外一個 Colab 的小事:每一格叫做一個 cell。你可以在每一格只寫一兩行,不需要把所有東西擠在同一格。右鍵可以插入新的 cell、可以複製、可以刪。把讀資料的放一格,把分析的放一格,這樣出錯的時候你可以一格一格 debug,不會整個亂掉。

Stack Overflow 也還是值得看。我知道現在有了 AI 你可能覺得不需要。但 AI 有時候給你一個可以跑的解,可是那個解有點怪——可能是五年前的寫法,或者社群早就決議不要這樣用。這種事 AI 不一定會主動告訴你。你覺得怪的時候,去 Stack Overflow 搜那個關鍵字,看一下社群的討論脈絡。不是每次都要看,但有那個習慣會讓你知道你用的 code 是不是主流的做法。

現在給你三個第一週可以做的 mini project,每個大概十到三十分鐘就能做完。第一天:去 Kaggle 下載 Titanic 的 dataset,讀進 Colab,看前五行,然後算一下男女的平均年齡。第三天:把 Titanic 的資料依照艙等分組,看每個艙等的生存率。第七天:自己用 Excel 隨便輸入三十個假病人的假資料,欄位你自己設計,把它讀進 Colab,做一個 table 1。不是要你去報線上課程,不是要你買書。就是這三個任務,全部用你自己的問題驅動。

在你開始之前,有幾個坑我先報給你知道。第一,不要第一天就想做 AI 或 deep learning。那是進階的事。第一週的目標只有一個:讀到自己的資料,看到前五行。就這樣。第二,不要跟著 YouTube 教學從頭看到尾。你會忘記,而且那些範例跟你沒有關係,你很快就不想看了。永遠跟著你自己想解決的問題走。第三,出錯是常態。[laughs] 我到現在每天還是在看 error message,那不代表你程度不好,那只代表你在寫 code。第四,如果你某天試著在自己電腦裝 Python 裝到出問題,那不是 coding 的問題,那是環境的問題——是兩件完全不同的事。別讓那個動搖你對自己 coding 能力的信心。這也是為什麼第一週我叫你用 Colab,就是讓你跳過這個坑。

最後提一件事當你這週結束的時候可能會有的感覺。當你開始覺得 Colab 的連線會斷很煩、或者你想跑一個比較長的東西讓它跑著你去做別的事、或者你想把你的 code 存在自己電腦上——那個感覺出現的那一天,是你畢業的訊號。那時候你會需要認識三樣東西:Miniconda,用來管理你電腦上的 Python 版本;conda env,讓每個專案有獨立的環境互不干擾;還有 Git,讓你的 code 有版本記錄、不會改壞了救不回來。這三樣不是今天的事。Git 的入門如果你怕命令列,先裝一個叫 GitHub Desktop 的視覺化 app,點點滑鼠就能做基本的事。這三樣我找一集專門講。
