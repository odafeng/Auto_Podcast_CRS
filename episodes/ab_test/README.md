This folder holds side-by-side script outputs for comparing prompt versions.
The protocol is documented in `docs/prompt_ab_testing.md`.

## How to run the blind listening comparison

### 1. Run TTS on both scripts

```bash
# v1 — re-render the shipped script into a fresh MP3
cp ../01_why-surgeons-should-code/script.txt /tmp/v1_ep01.txt
# Then generate MP3 via scripts/run_tts.py or manual ElevenLabs call

# v2 — same thing
cp 01_why-surgeons-should-code_v2_generic.txt /tmp/v2_ep01.txt
```

Budget: ~$0.80 total for both renders.

### 2. Rename to neutral files

```bash
mv v1_ep01.mp3 a.mp3
mv v2_ep01.mp3 b.mp3
```

Do NOT peek. Which is `a` and which is `b` should be hidden from you
until after you've rated both.

### 3. Listen end-to-end with headphones

For each file, rate 1–5 on:
- **Naturalness** — sounds like a person vs sounds like a narrator
- **Clarity** — is the core message coming through
- **Energy** — sustained engagement vs drifts into monotone

### 4. Record ratings + decide

See `docs/prompt_ab_testing.md` for the full decision rule.
Write results to `ab_test_ep01_2026-NN-NN.yaml` in this folder.

## Notes on v2 length

The v2 output came in at 2016 chars, below the prompt's target range of
2800–3500. This is expected — conversational style uses fillers and
tangents that reduce information density per character. If the v2 output
feels too short when listened to, either:

1. Lower the target in `system_v2_generic.md` to 2000–3000, or
2. Add more source material (more stories, more concrete examples)
   to give v2 more to work with

Do NOT pad the script with filler just to hit a character count.
Natural length > target length.

## Do not delete

These files are reference artifacts. Even after v2 becomes the default,
keep both v1 and v2 outputs for each evaluated episode. They're the
reproducibility record of the A/B decision.
